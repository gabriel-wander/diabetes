# Checklist de segurança clínica — Diabetes Pro BR

- [x] Exibir aviso de que o app é apoio técnico e não substitui julgamento médico.
- [x] Não enviar dados sensíveis para servidores.
- [x] Salvar casos apenas localmente e apenas com consentimento explícito do usuário/host (`explicitConsent: true`).
- [x] Mostrar classificação, justificativa, recomendação, alertas, referência, versão e data da regra.
- [x] Sinalizar HbA1c potencialmente incongruente quando houver anemia, hemoglobinopatia, gestação/puerpério, transfusão, DRC, eritropoetina, antirretrovirais ou similares.
- [x] Pedir confirmação quando apenas um critério laboratorial de diabetes estiver alterado.
- [x] Bloquear metas gestacionais no MVP e orientar consulta à diretriz específica.
- [x] Alertar que a escolha farmacológica no DM2 depende de TFG, contraindicações, custo e disponibilidade.
- [x] Nunca recomendar suspender insulina basal no DM1.
- [x] Alertar risco de CAD no DM1.
- [x] Acionar emergência nos dias de doença para vômitos persistentes >2h, alteração neurológica, cetonemia >1,5 mmol/L/cetonúria alta, dor abdominal importante, respiração de Kussmaul, desidratação, incapacidade de manter glicemia >70 mg/dL, criança <5 anos ou família incapaz de manejo domiciliar.
- [x] Tratar insulina como medicamento de alto risco no módulo de técnica segura.
- [x] Sinalizar desintensificação/simplificação no idoso frágil, com hipoglicemia, multimorbidade, cognição comprometida ou baixa expectativa de vida.
- [x] Manter catálogo JSON local versionado para auditoria dos parâmetros críticos implementados.
- [ ] Revisão clínica formal por especialistas antes de uso em ambiente assistencial.
- [ ] Validação prospectiva/usabilidade com público-alvo.
- [ ] Definição institucional de protocolos configuráveis para doses extras em dias de doença.
